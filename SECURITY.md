# Security Policy

## Supported Versions

Security updates are currently focused on the active development branch of Deadline Guardian.

| Version | Supported |
| :--- | :--- |
| v1.0.x (Current Hackathon Release) | :white_check_mark: Yes |
| < v1.0.0 | :x: No |

## Reporting a Vulnerability

We take the security of Deadline Guardian very seriously. If you find any security vulnerabilities or concerns, please do not open a public issue. Doing so invites exploitation before a fix can be applied.

Instead, please follow these steps:

1.  **Draft a detailed report** outlining the vulnerability. Include:
    *   Type of issue (e.g. CSRF, XSS, token exposure, credential leaking).
    *   Step-by-step instructions or PoC (Proof of Concept) code to replicate.
    *   Affected routes or components.
2.  **Send the report via email** to `saishivasanjeethpaikarao@gmail.com`.
3.  Our team will acknowledge receipt of your report within **48 hours** and coordinate a resolution window.
4.  Once a patch is released on `main`, credit will be attributed in the release notes unless you request to remain anonymous.
